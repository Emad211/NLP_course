2 Part 1: TF-IDF Theory — سلول‌های 7 تا 23

💡 رویکرد آموزشی: اول دستی، بعد کتابخانه در این بخش TF-IDF روی یک corpus کوچک ۴ جمله‌ای فارسی به‌صورت دستی محاسبه می‌شود. سپس با sklearn تأیید می‌شود. این رویکرد تضمین می‌کند دانشجو ریاضیات پشت الگوریتم را می‌فهمد، نه فقط API.

سلول 7 Markdown — مقدمه

معرفی TF-IDF به‌عنوان یکی از پایه‌ای‌ترین روش‌های IR (از دهه ۱۹۷۰). سه هدف این بخش.

سلول 8 Markdown — 1.1 The Core Problem

فرمول‌بندی مسئله: تابعی که متن را به بردار عددی تبدیل کند. نوتیشن ریاضی: $f: \text{text} \to \mathbb{R}^{|V|}$. مثال ۳ دکتر با query «سنگ کلیه».

سلول 9–10 Markdown + Code — 1.2 Term Frequency

فرمول TF + محاسبه دستی روی ۴ سند. خروجی: ماتریس TF به‌صورت DataFrame.

سلول 11 Markdown — Problem with Raw TF

دو ضعف بنیادی raw TF: (1) document length bias (2) عدم تمایز بین کلمات رایج و نادر. این بخش انگیزه معرفی IDF را ایجاد می‌کند.

سلول 12–13 Markdown + Code — 1.3 IDF

فرمول smooth IDF (همان فرمول sklearn) + توضیح نقش لگاریتم + smoothing. محاسبه دستی + نمایش informative vs non-informative terms.

سلول 14–15 Markdown + Code — 1.4 TF × IDF

ضرب دو مؤلفه + ماتریس TF-IDF. ۳ سطح scoring: rare+relevant → high, common → moderate, absent → zero.

سلول 16–17 Markdown + Code — 1.5 Sublinear TF

فرمول $1 + \log(\text{tf})$ + جدول compression ratio + نمودار مقایسه. چرا مهم: دکترهای پرنظر بدون sublinear TF نتایج را dominate می‌کنند.

سلول 18–19 Markdown + Code — 1.6 L2 Norm + Cosine Sim

فرمول L2 + فرمول cosine + چرا cosine بهتر از Euclidean (direction vs magnitude). محاسبه دستی query «سنگ کلیه» → ranking ۴ سند.

سلول 20–21 Markdown + Code — 1.7 Verification

sklearn TfidfVectorizer → مقایسه عدد به عدد با محاسبه دستی → ✓ Match.

سلول 22–23 Markdown + Code — 1.8 Bigrams

چرا unigram کافی نیست: «سنگ صفرا» ≠ «سنگ کلیه». دمو: unigram vs bigram scoring.

✅ در پایان Part 1، دانشجو: TF-IDF را از صفر ساخته، با sklearn تأیید کرده، و می‌داند چرا sublinear TF، L2 norm، و bigram لازم‌اند.

8 Part 7: Reranking — سلول‌های 49 تا 53

سلول 49 Markdown — The Problem + Formula

۳۷ خط توضیح علمی:

  • مسئله: Similarity ≠ Quality — دکتری با نظرات منفی مرتبط نباید بالا باشد
  • فرمول: $\text{final} = \text{sim} \times \text{smooth\_rec} \times \log(1 + \text{support})$
  • Bayesian Smoothing: چرا Beta(1,1)؟ جدول عددی: 1/1=100% vs 45/50=90% → بعد smoothing: 67% vs 88%
  • log(1+support): چرا لگاریتمی؟ ۱۰۰ نظر نباید ۵۰× بهتر از ۲ نظر باشد

سلول 50–51 Code — Evidence + Matching

build_evidence(): هر دکتر → لیست نظرات + آیا rate≥4. comment_matches(): بررسی حضور ترم کوئری (token-boundary safe).

سلول 52 Code — rerank()
